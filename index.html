Abstract

Motivation & approach

Modern computer-aided design (CAD) workflows still require significant manual effort to translate sketches and textual specifications into parametric models. This manual process limits rapid iteration and makes downstream automation difficult. CoCAD investigates a multimodal generative approach that maps images, natural language, and parametric CAD command sequences into a shared latent representation, enabling the automatic synthesis of valid CAD programs.

A sequence autoencoder is first trained to represent CAD modeling histories as compact latent codes while preserving command structure, constraints and parameter values. Visual sketches or rendered views are embedded using a frozen DINOv2 backbone, and textual descriptions are embedded with a frozen BERT encoder. Both modalities are projected into the CAD latent space through lightweight MLP adapters and a transformer-based fusion layer that aligns multimodal information.

On top of these representations, CoCAD employs a latent flow-matching generative model that learns to transport noise samples to the CAD latent distribution conditioned on image and text embeddings. During inference, the flow-matching model generates CAD latents that are decoded by the autoencoder into command sequences. The resulting outputs are valid, editable CAD programs that can be directly loaded into parametric CAD environments, providing a step toward end-to-end, multimodal automation of CAD modeling workflows.

Architecture

Model components & training pipeline

Architecture Summary

  • Autoencoder for CAD command sequence encoding and decoding
  • Frozen DINOv2 backbone for high-level image embeddings
  • Frozen BERT encoder for textual specification embeddings
  • MLP adapters and a transformer fusion layer for modality alignment
  • Latent generative model trained with flow matching in CAD space

The architecture separates representation learning for CAD and perception from the conditional generative model, allowing the use of strong pretrained encoders while keeping CAD-specific components compact and controllable.

Dataset

Multimodal CAD corpus

The CoCAD dataset is built from parametric CAD design histories, where each sample comprises a sequence of modeling commands, associated sketches, and rendered views of the final geometry. The command sequences are normalized into a consistent grammar and tokenization scheme suitable for sequence modeling, while parameter values and constraints are preserved to keep the designs editable.

For the multimodal setup, each CAD program is paired with one or more 2D inputs: rasterized sketch images capturing the design intent and rendered views of the final 3D model from multiple perspectives. Textual annotations summarize geometric features, functional roles, and design constraints. The dataset is split into training, validation, and test sets to avoid component-level leakage and to evaluate generalization to unseen geometries and descriptions.

Dataset Views

  • Parametric CAD command sequences with parameters & constraints
  • Sketch-like 2D inputs used as conditioning images
  • Multi-view renders of final models (front / side / isometric)
  • Short textual descriptions of function and geometry
  • Train/validation/test splits respecting design families
